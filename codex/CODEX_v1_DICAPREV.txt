## 3. Reglas de Diseño Premium

### Tipografía
- Inter (general)
- Display serif para títulos largos

### Bordes
- **rounded-2xl** por defecto
- Inputs: rounded-lg
- Modales: rounded-3xl

### Colores DICAPREV
- **Verde corporativo**: #00A86B
- **Verde oscuro**: #007A52
- **Naranjo botones premium**: #FF7A00
- **Grises UI**:
  - #F5F5F5 (fondos)
  - #E5E5E5 (bordes)
  - #222 (texto headings)

### Sombras Premium
- shadow-[0_4px_20px_rgba(0,0,0,0.08)]

### Transiciones
- Todas las interacciones: transition-all duration-200

## 4. Helpers y Utilidades Oficiales

### formatDate()
Convierte fechas para evitar hydration mismatch.

### estadoClase()
Asigna colores según el estado del documento:
- Vigente → verde
- Por vencer → amarillo
- Vencido → rojo

### getVigencia()
Reglas para calcular estados según fecha de vencimiento.

## 5. Convenciones Internas

### Nombres de Archivos
- page.tsx para páginas
- Modal + Nombre.tsx para modales
- Card + Componente.tsx para tarjetas
- use + Nombre.ts para hooks

### Normalización
- Ningún componente debe tener estilos inline
- Todo debe usar Tailwind + clases comunes
- No se duplican componentes
- Cada cambio grande se integra al CODEX
